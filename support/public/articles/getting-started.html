Getting started with Todoer

Last updated May 2025

Todoer is a task management app built around workspaces, lists, and collaboration. This guide covers everything you need to get up and running in the first few minutes.

Creating your account

Head to __APP_DOMAIN__/register and sign up with your email address. You'll receive a verification email — click the link inside to activate your account before you can sign in.

If you'd prefer to sign in with Google, use the Continue with Google button on the sign-in page. Google accounts don't require a separate email verification step.

Tip If you don't see the verification email within a few minutes, check your spam folder. The email comes from the Todoer address configured for your installation.

Your first workspace

After signing in you'll land on the Workspaces page. A workspace is a shared space for a project or team — everything in Todoer lives inside one. Click New workspace to create your first one.

Give the workspace a name and optional description. You can change both at any time from the workspace settings page.

Adding tasks

Inside a workspace, the main view is your task list. Click the input field at the top of the list and type to add a new task. Press Enter or click the add button to save it.

Each task supports:

  • Priority levels — Low, Medium, High, and Urgent
  • Due dates — click the calendar icon on any task to set one
  • Completion toggle — check the circle on the left to mark a task done

Inviting collaborators

Workspaces are more useful with your team. Open the workspace settings and navigate to the Members tab to invite people by email. Invited users receive an email with a join link.

See Inviting collaborators to a workspace for the full details.

Uploading files

Todoer supports file attachments at the workspace level. Use the Files tab inside any workspace to upload documents, images, and other assets. Supported formats and size limits are covered in How file uploads work.

The public feed

Each workspace has an optional public feed — a read-only view of completed tasks that anyone can see without signing in. This is useful for public project updates or changelogs. See Using the public feed for setup instructions.
