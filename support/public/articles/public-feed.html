Using the public feed

Last updated May 2025

The public feed is an opt-in, read-only view of a workspace's completed tasks. It's accessible to anyone without signing in — useful for public project updates, open source changelogs, or client-facing progress reports.

Enabling the public feed

  1. Open workspace settings via the gear icon
  2. Go to the Public Feed tab
  3. Toggle Enable public feed on
  4. Copy the public URL that appears

The feed is immediately accessible at the URL shown. No additional configuration is required.

What the feed shows

The public feed displays completed tasks in reverse chronological order. Each entry shows:

  • Task title
  • Priority label
  • Completion date
  • The display name of the member who completed it

Tasks that are pending or in progress are not shown. Deleted tasks do not appear even if they were previously completed.

Privacy note Member display names are visible in the public feed. Make sure your team is comfortable with this before enabling the feature on a workspace with sensitive task titles.

Disabling the public feed

Return to the Public Feed tab in workspace settings and toggle it off. The URL immediately stops working — anyone who tries to access it will receive a 404 response.

Feed URL

The feed URL follows the pattern __APP_DOMAIN__/feed/[workspace-id]. The workspace ID is a randomly generated identifier — it cannot be guessed, but it should still be treated as semi-private. Don't share it beyond your intended audience.
