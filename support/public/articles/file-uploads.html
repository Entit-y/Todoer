Files & Uploads

How file uploads work

Last updated May 2025

Todoer supports file uploads at the workspace level. Files are shared across all members of the workspace and can be downloaded by anyone with access.

Uploading a file

  1. Open a workspace and click the Files tab
  2. Click Upload file or drag and drop files onto the upload area
  3. Wait for the upload to complete — a progress indicator will appear for larger files

Uploaded files appear in the list immediately. Each entry shows the filename, uploader, file size, and upload date.

Supported file types

Todoer accepts most common file types including documents, images, archives, and spreadsheets. Executable files (.exe, .sh, .bat) are blocked at upload.

ZIP archives are accepted and their contents are extracted automatically for preview. The extracted files are stored alongside the original archive.

Note ZIP extraction is performed server-side. Very large archives or archives with deeply nested paths may take a moment to process.

File size limits

Individual file uploads are limited to 50 MB. If you need to share something larger, consider splitting the archive or using an external file sharing service and pasting the link into a task.

Downloading and deleting files

Click any file in the list to download it. To delete a file, hover over it and click the trash icon. Workspace owners and the original uploader can delete files — other members cannot.

Deleted files are permanently removed and cannot be recovered.

File visibility

Files are visible to all workspace members. There is currently no per-file permission control — if a user has access to the workspace, they can see and download all files in it.

Files are not exposed through the public feed, even if the workspace's public feed is enabled.
